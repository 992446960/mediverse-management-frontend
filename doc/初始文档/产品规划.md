## 一、产品定位

**产品名称：** Mediverse Management  
**一句话定位：** 面向医疗机构的可视化数字医生分身与知识库管理平台——让每位医生拥有一个可成长的 AI 分身，让每个机构的医学知识可沉淀、可流通、可复用。

### 核心价值主张

| 维度               | 价值                                             |
| ------------------ | ------------------------------------------------ |
| 对医生（普通用户） | 将个人临床经验和知识体系结构化，训练专属 AI 分身 |
| 对科室/机构管理者  | 管理机构知识资产，监控数字医生分身使用情况       |
| 对系统管理员       | 全局运维、多租户管理、API 授权、会话溯源         |

## 三、核心概念模型

```
机构（Organization）── 全科数字医生（Org Digital Doctor）/ 机构知识库
  └── 科室（Department）── 科室数字医生（Dept Digital Doctor）/ 科室知识库
        └── 用户（User）
              ├── 数字医生分身（Digital Doctor Avatar）◄──────────────┐（按绑定范围访问）
              │     ├── 会话记录（Sessions）                          │
              │     │     └── 消息（Messages）                       │
              │     └── 统计数据（Stats）                             │
              │                                                       │
              └── 知识库（Knowledge Base）────────────────────────────┘
                    ├── 知识文件（Files）
                    │── 知识卡（Knowledge Cards）
                    └── 统计数据（Stats）
```

> 知识文件处理后生成知识卡，存放在知识库中。分身根据绑定范围（机构/科室/用户）访问对应的知识库。

### 数字医生分身类型

| 类型         | 绑定对象         | 说明                                       |
| ------------ | ---------------- | ------------------------------------------ |
| 全科数字医生 | 机构             | 具备全科医学知识体系，绑定机构级知识库     |
| 专科数字医生 | 科室             | 具备特定专科深度知识，绑定科室级知识库     |
| 专家分身     | 用户（医生个人） | 复现特定专家的诊疗风格、个人知识与经验积累 |

**分身的三种使用模式（适用于所有类型）：**

- **面向患者（医生分身模式）：** 分身代替医生接诊，为患者提供专业问诊、诊断与健康咨询
- **面向医生（助手模式）：** 分身作为医生的 AI 工作助手，提供病情评估、诊断建议、文献检索、决策辅助、病历撰写等专业支持
- **面向系统（监控巡查模式）：** 分身 7×24 小时监听系统事件（如新的体征、检查报告数据等），持续对患者状态进行跟踪和自动评估，在需要预警或干预时主动通知医生

### 知识处理流水线

```
文件上传 → 解析（文本提取）→ 结构化知识抽取 → 索引入库 → 可被分身引用
```

### 知识卡类型

| 类型   | 说明                                 |
| ------ | ------------------------------------ |
| 循证卡 | 来自指南/文献的循证医学证据          |
| 规则卡 | 临床决策规则（如诊断标准、用药规范） |
| 经验卡 | 临床经验与个人实践总结               |

## 四、模块功能详述

---

### 模块 0：仪表盘

**访问权限：** 所有已登录用户  
**核心功能：**

- 累计接诊数、今日接诊数
- 累计使用 Token、今日使用 Token
- 累计引用知识次数

### 模块 1：机构管理

**访问权限：** 仅系统管理员  
**核心功能：**

- 新增/编辑/删除机构
- 分页查询机构列表
- 机构基础信息：名称、机构编码、描述、Logo、状态（启用/禁用）

**关键字段建议：**

```
机构（Organization）
- id, name, logo_url
- code（机构编码）
- description（描述）
- status（active/inactive）
- created_at, updated_at
```

---

### 模块 2：科室管理

**访问权限：** 系统管理员、机构管理员  
**核心功能：**

- 左侧机构目录树（按机构名分组）
- 新增/编辑/删除科室
- 分页查询科室列表

**交互设计要点：**

- 目录树支持折叠/展开
- 机构管理员只能看到自己所属机构的科室

**关键字段：**

```
科室（Department）
- id, org_id（所属机构）
- name, code（科室编码）
- description
- status（active/inactive）
- created_at, updated_at
```

---

### 模块 3：用户管理

**访问权限：** 系统管理员、机构管理员、科室管理员  
**核心功能：**

- 左侧机构-科室目录树
- 新增/编辑/删除用户
- 分页查询用户列表
- 设置用户角色
- 账号/密码管理（重置密码）
  **角色隔离规则：**
- 科室管理员只能管理本科室用户
- 机构管理员只能管理本机构用户

**关键字段：**

```
用户（User）
- id, org_id, dept_id
- username, password_hash
- real_name, avatar_url, phone, email
- remark（备注）
- roles（[sysadmin | org_admin | dept_admin | user]）
- status（active/inactive）
- created_at, updated_at
```

### 模块 4：数字医生体验

**访问权限：** 所有已登录用户  
**核心功能：**

#### 4.1 分身选择

- 新建会话时按机构-科室筛选并选择分身
- 展示分身头像、名称、类型标签、简介

#### 4.2 会话管理

- 左侧会话列表按时间分组（今日、昨天等）
- 新建 / 重命名 / 删除会话
- 点击会话进入对应对话窗口

#### 4.3 对话窗口

- 三栏布局：左侧会话列表 + 中间对话区 + 右侧技能区
- 输入区支持：思考模式（高/中/低）选择、联网搜索开关
- 文本输入 + 语音输入（ASR）
- 上传附件（图片、PDF、文档）
- 流式打字机效果（SSE/WebSocket）
- 思考过程折叠展示（timeline 形式：step 名称与结果描述）
- 技能区：点击技能展示执行过程和结果

#### 4.4 评分与反馈

- 星级评分维度：诊断准确率、问诊完成度（可扩展）
- 文本反馈（可选填）
- 评分时机：**用户主动点击评分按钮触发**（非自动弹出）

**关键字段：**

```
会话（Session）
- id, user_id, avatar_id
- title（默认取首条消息截断，可重命名）
- status（active/archived）
- created_at, updated_at

消息（Message）
- id, session_id, avatar_id
- role（user | assistant | tool）
- parts（JSON数组，支持同一条消息同时包含多种内容）
  - { type: "text", text: "..." }
  - { type: "image", url: "...", file_name: "..." }
  - { type: "file", url: "...", file_name: "...", mime_type: "..." }
- tool_calls（JSON，工具调用记录，role=tool 时使用）
- tokens_used
- created_at

评分（Rating）
- id, session_id, user_id
- scores（JSON，对象形式存储各评分维度，如 { accuracy: 4, completion: 5 }）
- feedback_text
- created_at
```

### 模块 5：数字医生分身管理

**访问权限：** 系统管理员、机构管理员、科室管理员  
**核心功能：**

- 创建分身：选择类型 → 绑定机构/科室/用户 → 上传头像 → 填写基础信息
- 编辑/删除分身
- 分页列表（支持按类型、机构、科室过滤）

**创建流程（向导式）：**

```
Step 1: 选择类型（全科 / 专科 / 专家分身）
Step 2: 绑定范围（机构 / 科室 / 用户）
Step 3: 基础信息（名称、头像、简介、标签）
Step 4: 确认创建
```

**关键字段：**

```
数字医生分身（Avatar）
- id, type（general | specialist | expert）
- org_id（全科必填）, dept_id（专科必填）, user_id（专家分身必填）
- name, avatar_url, bio, tags（JSON数组）
- greeting（开场白）, style（沟通风格）
- status（active/inactive）
- created_by, created_at, updated_at
```

### 模块 6：我的分身

**访问权限：** 已创建分身的用户  
**核心功能：**

#### 6.1 分身配置

- 个人简介（富文本）
- 个人标签（技能标签，通过加号图标添加，支持移除）
- 开场白（对话开始时自动发送的欢迎语）
- 沟通风格（正式/亲切/简洁/详尽等预设选项 + 自定义描述输入框）

- 点击「对话测试」进入独立测试页面
- 与自己的分身对话，对话窗口与数字医生体验一致（思考模式、联网搜索等）
- 测试页面不展示会话列表和技能区，专注对话验证

#### 6.3 统计看板

| 指标       | 说明                           |
| ---------- | ------------------------------ |
| 累计接诊数 |                                |
| 今日接诊数 |                                |
| 学习进度   | 已索引文件 vs 上传的文件总数量 |

### 模块 7：我的文件

**访问权限：** 已创建分身的用户  
**核心功能：**

#### 7.1 目录管理

- 左侧目录为树形结构
- 「所有文件」与「未分类」为两个并列独立入口
- 点击「所有文件」按时间倒序列出全部文件
- 默认 6 个一级目录：**指南 / 共识 / 文献 / 病例 / 规则 / 其他**
- 支持在一级目录下创建多级子目录
- 目录重命名/删除

#### 7.2 文件上传与管理

- 支持格式：PDF、TXT、CSV、JSON、MD、XLSX、DOCX
- 上传时可选择：自动分类（AI 推荐）或手动选择分类
- 上传后自动处理流水线：
  ```
  上传中 → 解析 → 知识抽取 → 索引 → 完成
  ```
- 文件状态标记：处理中 / 已完成 / 失败（可重试）
- 表格列：名称、格式、大小、知识卡数量、发布机构、发布时间、操作
- 操作：预览文件、下载、删除

#### 7.3 文件预览

- 预览在页面内完成，不打开新窗口
- 左侧大窗口支持切换：PDF 原文视图 / 解析后文档
- 右侧展示从该文件抽取的知识卡（循证卡/规则卡/经验卡）

**关键字段：**

```
目录（Directory）
- id, owner_type（personal | dept | org）, owner_id
- parent_id（null 为一级目录）, name, sort_order
- is_default（是否为系统默认目录）

文件（File）
- id, owner_type, owner_id, dir_id
- file_name, file_type, file_size, storage_url
- status（uploading | parsing | extracting | indexing | done | failed）
- auto_category_suggestion（AI 推荐目录 id）
- created_by, created_at, updated_at
```

### 模块 8：我的知识卡

**访问权限：** 已创建分身的用户  
**核心功能：**

#### 8.1 知识卡列表

- 按类型分 Tab 展示：循证卡 / 规则卡 / 经验卡
- 表格列：标题、类型、状态、引用次数、更新时间、操作
- 支持搜索、过滤（状态、来源文件、标签）
- 创建、编辑卡片功能可响应并更新数据

#### 8.2 版本管理

- 每次编辑自动生成新版本
- 版本 timeline（时间 + 操作人 + 摘要）
- 可预览历史版本、回退至指定版本（预览与回退为两个独立按钮）
- 内容支持编辑，保存时生成新版本

#### 8.3 状态管理

- 上线/下线（控制分身是否可引用）

#### 8.4 知识卡详情

- 卡片内容全文展示（可编辑）
- 溯源文件列表（点击可跳转到原文件对应位置）
- 版本更新 timeline（预览、回退操作）
- 引用次数统计

**关键字段：**

```
知识卡（KnowledgeCard）
- id, owner_type（personal | dept | org）, owner_id
- type（evidence | rule | experience）
- title, content（Markdown）
- tags（JSON数组）
- online_status（online | offline）
- audit_status（pending | approved | rejected）
- current_version, reference_count
- created_by, created_at, updated_at

知识卡来源（KnowledgeCardSource）
- id, card_id, file_id
- page_hint（可选，页码/位置提示）
- created_at

知识卡版本（KnowledgeCardVersion）
- id, card_id, version_number
- content（该版本完整内容）
- change_summary
- operated_by, created_at
```

### 模块 11：知识库搜索

**访问权限：** 所有已登录用户  
**核心功能：**

#### 11.1 搜索入口页

- 搜索框文案：「搜索全部」，支持自然语言提问（最多 500 字）。
- 展示最近 N 条搜索历史（从 `sessions` 中 `type = 'kb_search'` 的会话取标题/问题）。
- 下方卡片展示：最新文件、最新知识卡。
- 在入口页发起搜索时创建新的知识库搜索会话并跳转 `/knowledge-base/search/{session_id}?q=...`。

#### 11.2 搜索会话列表

- 左侧展示当前用户的知识库搜索会话列表（`sessions.type = 'kb_search'`），按时间分组（今天/昨天/更早）。
- 展示内容：会话标题、最近更新时间。
- 支持新建、重命名、删除搜索会话。

#### 11.3 搜索结果与多轮对话

- 中间区域以多轮线程形式展示：每轮包含一条用户消息和一条 AI 消息（复用 `messages` 表）。
- AI 消息包含：
  - 综合回答正文（Markdown）
  - 引用来源列表（文件/知识卡 Sources）
  - 思考过程（`ThinkingStep[]`，用于 ThinkingProcess 组件）
  - 相关问题推荐（Related Questions）
- 页面底部输入框支持继续提问，追加为同一会话的下一轮消息，保留上下文。

**搜索范围：** 用户有权访问的所有知识库（个人 + 科室 + 机构）

**关键字段：**（复用通用对话模型，增加场景标记）

```
会话（Session）
- id, user_id
- type（chat | kb_search）
- title（默认取首轮问题截断，可重命名）
- created_at, updated_at

消息（Message）
- id, session_id
- role（user | assistant）
- parts（文本为主）
- thinking_process（ThinkingStep[]，用于展示思考过程）
- sources（可选，用于记录本轮引用的文件/知识卡）
- created_at
```

### 模块 12：API Token 管理

**访问权限：** 仅系统管理员  
**核心功能：**

- 创建 Token（填写名称/描述/所属机构，系统自动生成 Token 字符串）
- 查看 Token 列表（名称、Token 码、创建时间、最后使用时间、状态）
- Token 明文默认隐藏，点击小眼睛图标可显示/隐藏
- 支持复制 Token 到剪贴板
- 启用/禁用/删除 Token
- 编辑 Token 时不可修改所属机构

**关键字段：**

```
API Token
- id, name, description
- org_id（所属机构，编辑时不可修改）
- token
- status（active/inactive）
- last_used_at
- created_by, created_at
```

### 运维监控模块（Other，仅系统管理员）

#### 任务管理

- 查看文件处理任务队列（按状态/机构/时间筛选）
- 手动重试失败任务
- 任务详情：执行耗时、错误信息

**关键字段：**

```
任务（Task）
- id, type（file_parse | knowledge_extract | index）
- ref_id（关联对象 id）, ref_type
- status（pending | running | done | failed）
- error_msg, retry_count
- started_at, finished_at, created_at
```

#### 会话/消息管理

- 分页查询所有会话（按机构/科室/分身/用户/时间过滤）
- 查看单条会话完整消息流
- 查看消息中的知识卡引用详情
- 查看 SKILL 调用记录（调用参数、返回结果、耗时）

#### Token 用量统计

- 按机构 / 科室 / 用户 / 分身统计 LLM Token 消耗
