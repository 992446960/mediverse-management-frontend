## 一、整体布局

### 1.1 全局框架（已登录状态）

```
┌─────────────────────────────────────────────────────────────┐
│  LOGO  │  导航菜单（按角色动态渲染）              │ 用户信息 ▼│  ← 顶部栏（高度 56px）
├────────┴──────────────────────────────────────────────────-─┤
│        │                                                     │
│  侧边栏 │                 主内容区                            │
│ (240px)│                                                     │
│        │                                                     │
└────────┴─────────────────────────────────────────────────────┘
```

**顶部栏：**
- 左：产品 Logo + 名称（Mediverse）
- 中：一级导航 Tab —— 依角色显示
- 右：通知铃铛图标 | 用户头像 + 姓名下拉（包含"个人设置"/"退出登录"）

**左侧边栏：**
- 左上角：用户所属机构（固定显示，不提供切换）
- 平台功能：仪表盘、我的工作台、科室工作台、机构工作台、数字医生体验、知识库搜索
- 系统管理：机构管理、科室管理、用户管理、分身管理、API Token、系统运维
- 支持折叠（collapsed 宽度 64px，仅显示图标）
- 当前激活菜单项高亮

**主内容区：**
- 面包屑导航（二级以上页面显示）
- 页面标题 + 操作按钮区
- 内容主体

---

### 1.2 导航菜单结构（按角色）

**平台功能（菜单顺序）：**
1. 仪表盘
2. 我的工作台（我的分身、我的文件、我的知识卡）
3. 科室工作台（科室数字医生、科室文件、科室知识卡）
4. 机构工作台（机构数字医生、机构文件、机构知识卡）
5. 数字医生体验
6. 知识库搜索

**系统管理：**
- 机构管理、科室管理、用户管理、分身管理、API Token、系统运维

（各角色可见范围依权限矩阵控制，详见产品规划.md）

- Shift+Enter 换行，Enter 发送
- 输入框高度随内容自适应（最多显示 6 行）

- 左侧：搜索会话列表
- 主区：用户问题 + AI 回答（引用来源、思考过程、综合内容、相关追问）
- 底部：继续对话输入框

## 十一、关键交互流程

### 11.1 新建分身完整流程

```
点击"创建分身"
  → 打开步骤向导
  → Step 1: 选择类型 → 点击下一步
  → Step 2: 选择绑定范围 → 点击下一步
  → Step 3: 填写基础信息（名称必填）→ 点击下一步
  → Step 4: 确认信息预览 → 点击"确认创建"
  → loading 动画
  → 成功 Toast + 跳转至分身详情页（或关闭向导刷新列表）
```

### 11.2 文件上传完整流程

```
点击上传按钮 or 拖拽文件
  → 客户端格式/大小校验
    → 不通过: 显示错误提示，不上传
    → 通过: 显示上传进度条
  → 上传完成，文件出现在列表，状态"解析中"
  → 状态轮询（每 5 秒）or WebSocket 推送
    → 解析中 → 知识抽取中 → 索引中 → 完成 ✅
    → 失败 ❌: 显示错误原因 + 重试按钮
  → 完成后 Toast 提示"文件已处理完成，抽取了 N 个知识卡"
```

### 11.3 发起对话完整流程

```
分身选择页 → 点击分身卡片 → 创建新会话并跳转至对话窗口
  → 显示分身开场白
  → 用户输入消息并发送
    → 请求 API，开启 SSE 流式连接
    → 流式接收 token，实时渲染
    → 思考过程折叠展示
    → 引用知识卡上标注释
    → 生成完毕，输入框可用
```
