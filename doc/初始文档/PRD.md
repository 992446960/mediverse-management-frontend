## 一、背景与目标

### 1.1 问题陈述

医疗机构中大量高质量的临床知识分散在个人医生的脑海、零散的文件和口口相传中，难以沉淀和复用。医生无法随时借助自己积累的知识高效辅助诊疗，机构的集体知识资产也缺乏系统化管理手段。

### 1.2 产品目标（第一期）

- 让每个医疗机构能建立并管理三级知识体系（机构 / 科室 / 个人）
- 让每位医生能基于知识库创建并训练专属 AI 分身
- 让任意授权用户可通过数字医生分身进行专业问诊或获取决策建议
- 为系统管理员提供完整的多租户管理能力

### 1.3 范围说明

**第一期包含：** 模块 1–12  
**第一期不包含：** 知识共享（跨机构）、完整运维监控

## 二、用户故事与验收标准

### US-00：仪表盘

**用户故事：** 作为已登录用户，我希望在首页看到关键业务指标概览，以便快速了解平台使用情况。

**验收标准：**

- 展示：累计接诊数、今日接诊数、累计使用 Token、今日使用 Token、累计引用知识次数

---

### US-01：登录与认证

**用户故事：** 作为任意角色用户，我希望通过账号密码登录系统，以便访问我权限范围内的功能。

**验收标准：**

- 输入正确账号密码后跳转至首页，不同角色看到不同的导航菜单
- 密码错误提示"账号或密码错误"，~~连续失败 5 次锁定账号 15 分钟~~
- 登录状态通过 JWT Token 维持，Token 过期时间 8 小时，支持无感刷新
- 退出登录后 Token 失效，跳转至登录页
- 账号被禁用时提示"账号已停用，请联系管理员"

---

### US-02：机构管理

**用户故事：** 作为系统管理员，我希望管理机构信息，以便维护多租户基础数据。

**验收标准：**

- 可新增机构，必填：名称；选填：机构编码、描述、Logo
- 机构名称不可重复，重复时提示"机构名称已存在"
- 可编辑机构基本信息（除 id 外所有字段）
- 删除机构时，若该机构下存在科室或用户，需二次确认
- 列表支持按名称模糊搜索，默认按创建时间倒序
- 可对机构执行启用/禁用，禁用后该机构下所有用户无法登录

---

### US-03：科室管理

**用户故事：** 作为系统管理员或机构管理员，我希望管理科室信息，以便维护机构的组织架构。

**验收标准：**

- 左侧展示机构目录树；系统管理员可见所有机构，机构管理员仅见所属机构
- 点击机构节点，右侧展示该机构下的科室列表
- 科室需有科室编码字段
- 同一机构内科室名称不可重复
- 删除科室时，若科室下有用户，提示用户并要求确认
- 列表支持按名称模糊搜索，支持按状态过滤

---

### US-04：用户管理

**用户故事：** 作为管理员，我希望管理用户账号和角色，以便控制系统访问权限。

**验收标准：**

- 左侧展示机构-科室目录树，点击节点过滤对应用户列表
- 新增用户必填：用户名（唯一）、真实姓名、密码、所属机构、所属科室；选填：手机号、邮箱、备注
- 初始密码由系统生成，首次登录强制修改密码
- 支持为用户设置多个角色，同一用户可同时是机构管理员和科室管理员
- 科室管理员只能新增/编辑本科室用户，机构管理员只能操作本机构用户
- 支持重置密码功能

---

### US-05：数字医生分身管理

**用户故事：** 作为管理员，我希望创建和管理数字医生分身，以便提供可供体验的 AI 医生。

**访问权限：** 系统管理员、机构管理员、科室管理员

**角色隔离：**
- 科室管理员只能管理本科室下的分身（专科数字医生）及本科室用户的专家分身
- 机构管理员只能管理本机构下的分身（全科、专科、专家分身）
- 系统管理员可管理全部

**验收标准：**

- 全科数字医生必须绑定机构；专科数字医生必须绑定科室；专家分身必须绑定用户
- 同一绑定范围（机构/科室/用户）下分身名称不可重复
- 头像支持上传（JPG/PNG，最大 2MB），不上传则展示默认头像
- 分身列表支持按类型、机构、科室过滤，支持名称模糊搜索
- 分身被禁用后，用户无法选择该分身发起会话

---

### US-06：我的分身 - 配置

**用户故事：** 作为拥有专家分身的医生，我希望配置分身的个性化信息，以便让分身更好地表达我的专业风格。

**验收标准：**

- 可编辑：简介（富文本，最多 500 字）、标签（通过加号图标添加，最多 10 个，每个最多 20 字，支持移除）、开场白（最多 200 字）、沟通风格
- 沟通风格提供预设选项：正式 / 亲切 / 简洁 / 详尽，选完后展示风格描述输入框，可自定义修改
- 配置变更立即对新会话生效

---

### US-07：我的分身 - 对话测试

**用户故事：** 作为拥有专家分身的医生，我希望与自己的分身对话测试，以便验证分身的表现是否符合预期。

**验收标准：**

- 点击「对话测试」进入独立测试页面，对话窗口与数字医生体验一致（思考模式、联网搜索、流式输出等）
- 测试页面不展示会话列表和技能区，专注对话验证
- 测试会话不计入正式接诊统计数据
- 测试会话标记为"测试"，与正式会话分开存储

---

### US-08：我的分身 - 统计看板

**用户故事：** 作为拥有专家分身的医生，我希望查看分身的接诊统计，以便了解分身的使用情况。

**验收标准：**

- 展示：累计接诊数（所有正式会话总数）、今日接诊数（当日 0:00 起）、学习进度（已完成索引的文件 / 总量）

---

### US-09：知识文件管理（个人/科室/机构通用）

**用户故事：** 作为有权限的用户，我希望上传和管理医学文件，以便构建知识库。

**验收标准：**

- 左侧目录为树形结构，「所有文件」「未分类」为两个并列独立入口，点击「所有文件」按时间倒序列出全部文件
- 支持格式：PDF、TXT、CSV、JSON、MD、XLSX、DOCX
- 上传时可选择：自动分类（AI 推荐）或手动选择分类
- 单文件最大 50MB，单次批量上传最多 20 个文件
- 上传后自动进入处理流水线，状态依次流转：上传中 → 解析 → 知识抽取 → 索引 → 完成
- 处理失败的文件展示错误原因，提供"重试"按钮
- 文件表格列包含：名称、格式、大小、知识卡数量、发布机构、发布时间、操作
- 操作列提供：预览文件、下载、删除
- 预览在页面内完成，不打开新窗口；支持切换 PDF 原文视图 / 解析后文档
- 预览页右侧展示从该文件中抽取的知识卡（循证卡/规则卡/经验卡）
- 目录管理：默认 6 个一级目录不可删除，可在其下创建多级子目录；目录名最多 30 字
- 删除含文件的目录前须用户二次确认

---

### US-10：知识卡管理（个人/科室/机构通用）

**用户故事：** 作为有权限的用户，我希望管理知识卡，以便精细控制分身可使用的知识。

**验收标准：**

- 列表按类型分 Tab：循证卡 / 规则卡 / 经验卡
- 表格列包含：标题、类型、引用次数、状态、更新时间、操作
- 支持按上线状态、审核状态、标签过滤；支持标题/内容关键词搜索
- 知识卡内容支持富文本编辑，保存时自动生成新版本
- 创建、编辑卡片功能可响应并更新数据
- 查看详情：卡片内容可编辑，版本 timeline 支持预览、回退（两个独立按钮）
- 版本列表展示：版本号、操作人、操作时间、摘要
- 可回滚至历史任意版本（回滚操作本身也生成新版本）
- 上线/下线操作立即生效：下线后分身新会话中不再引用该卡
- 机构/科室知识卡支持审核流程：提交审核 → 已审核 / 已驳回；驳回时必须填写驳回原因

---

### US-12：数字医生体验 - 分身选择

**用户故事：** 作为用户，我希望浏览并选择数字医生分身，以便开始与 AI 医生对话。

**验收标准：**

- 新建会话时展示分身列表
- 分身卡片展示：头像、名称、类型标签（全科/专科/专家）、所属机构/科室、简介摘要
- 支持按名称搜索，支持按机构、科室筛选
- 点击分身卡片创建新会话并进入对话窗口

---

### US-13：数字医生体验 - 会话管理

**用户故事：** 作为用户，我希望管理与分身的对话会话，以便复盘历史对话或开启新话题。

**验收标准：**

- 左侧会话列表按时间分组（今日、昨天等）展示：标题、最近消息时间、分身名
- 支持新建会话（选择分身后创建）
- 支持重命名、删除会话（二次确认）
- 列表按最近消息时间倒序排列

---

### US-14：数字医生体验 - 对话

**用户故事：** 作为用户，我希望与数字医生进行自然语言对话，以便获得专业的医疗建议。

**验收标准：**

- 三栏布局：左侧会话列表 + 中间对话区 + 右侧技能区
- 输入区支持：思考模式（高/中/低）选择、联网搜索开关
- 支持文本输入（最多 2000 字/条）
- 支持语音输入，语音转文字后展示在输入框，用户可编辑后发送
- 支持上传附件（图片支持 JPG/PNG/WEBP，最大 10MB；文档支持 PDF/DOCX，最大 20MB），同一条消息可同时包含文本和多个附件
- AI 回复以流式打字机效果渲染，逐 token 显示
- 思考过程以 timeline 形式展示（step 名称与结果描述），可折叠，默认折叠
- 技能区：点击技能展示执行过程和结果
- SKILL 调用展示：SKILL 名称（带 tooltip 显示描述）、调用参数摘要、返回结果摘要
- AI 输出中的引用知识卡以上标注释展示，点击可在侧边栏查看知识卡详情
- 消息发送中显示发送状态，发送失败提示错误并支持重试
- 流式输出中途可点击"停止生成"按钮中断

---

### US-15：数字医生体验 - 评分与反馈

**用户故事：** 作为用户，我希望对分身的回答进行评分，以便帮助改进分身质量。

**验收标准：**

- 对话界面提供"评分"按钮，用户主动点击触发评分弹窗
- 评分维度：诊断准确率（1-5 星）、问诊完成度（1-5 星）
- 支持可选填的文本反馈（最多 500 字）
- 每个会话可评分多次，记录每次评分时间
- 评分提交成功后给出感谢提示

---

### US-16：知识库问答

**用户故事：** 作为用户，我希望通过自然语言搜索知识库，以便快速找到所需的医学知识。

**验收标准：**

- 搜索框文案：「搜索全部」，支持自然语言提问（最多 500 字）
- 展示最近 N 条历史搜索记录，点击可快速填充
- 首页下方卡片展示：最新文件、最新知识卡
- 搜索后进入子页面，左侧为搜索会话列表，布局参考 Perplexity
- 子页面展示：搜索到的文件/知识卡、AI 综合回答、引用来源、思考过程、相关追问
- 可继续追问，形成多轮对话；每轮保留上下文

---

### US-17：API Token 管理（用于开放平台）

**用户故事：** 作为系统管理员，我希望管理 API Token，以便安全地授权外部系统访问接口。

**验收标准：**

- 创建 Token 时填写名称（必填，唯一）、描述（可选）、所属机构
- 提供"复制"按钮
- Token 列表展示：名称、描述、Token 码（默认隐藏，点击小眼睛可显示）、创建时间、最后使用时间、状态
- 支持复制 Token 到剪贴板
- 支持启用/禁用 Token，禁用后使用该 Token 的请求返回 401
- 编辑 Token 时不可修改所属机构
- 删除 Token 需二次确认

---

### US-18：科室工作台 / 机构工作台

**用户故事：** 作为科室管理员或机构管理员，我希望在对应工作台下管理科室/机构级数字医生、文件和知识卡，以便完成本层级的知识与分身管理。

**验收标准：**

- 科室工作台：科室数字医生、科室文件、科室知识卡；页面顶部提供机构切换 + 科室切换（机构在科室前）
- 机构工作台：机构数字医生、机构文件、机构知识卡；页面顶部提供机构切换
- 科室/机构数字医生的统计指标、对话测试功能与「我的分身」一致
- 科室/机构文件、知识卡的功能验收标准与 US-09、US-10 一致，仅数据作用域不同

## 三、业务规则汇总


| 规则编号  | 规则描述                                           |
| ----- | ---------------------------------------------- |
| BR-01 | 用户只能访问自己所属机构的数据，系统管理员无此限制                      |
| BR-02 | 分身根据绑定范围（机构/科室/用户）访问对应知识库                      |
| BR-03 | 知识卡下线后，新会话不再引用；已引用的历史记录不受影响                    |
| BR-04 | 文件处理状态机为单向流转，失败时可重试，成功后状态不可回退                  |
| BR-06 | Token 明文存储于 api_tokens 表，创建时展示一次后可通过界面显示/隐藏 |
| BR-07 | 操作日志只增不改，不支持删除                                 |
| BR-08 | 机构/科室知识卡需审核后才能上线，个人知识卡无需审核                     |
| BR-09 | 科室管理员不可操作机构级知识库；普通用户不可操作科室/机构级知识库              |
| BR-10 | 专家分身只有绑定的用户本人可配置（"我的分身"入口）；管理员可在分身管理模块进行基础信息编辑 |

## 四、非功能需求

### 4.1 性能

- 页面首屏加载时间 < 2 秒（P90）
- 对话首 token 响应时间 < 3 秒（P90）
- 文件上传接口支持并发 10 个文件同时上传
- 列表查询接口响应时间 < 500ms（P95）

### 4.2 可靠性

- 系统可用性 ≥ 99.5%（月度）
- 文件处理失败后支持自动重试（最多 3 次），超出后标记为失败并通知用户
- 流式对话中断后，用户刷新页面可看到已生成的部分内容

### 4.3 安全性

- 所有接口需携带有效 JWT Token，否则返回 401
- 所有数据访问在后端强制校验 `org_id` 范围，防止越权访问
- 文件上传须进行类型白名单校验和内容安全扫描
- 敏感操作（删除机构/科室/用户）需记录操作日志
- HTTPS 全站启用，传输层加密

### 4.4 兼容性

- 支持 Chrome 100+、Safari 15+、Edge 100+ 最新两个大版本
- 最小支持分辨率：1280×720
- 对话页面在 1920×1080 及以上显示器上有优化展示

### 4.5 可维护性

- 前端组件库统一，避免同一功能多套实现
- 后端接口遵循 RESTful 规范
- 所有接口变更需更新 OpenAPI 文档

## 五、边界条件与异常处理


| 场景           | 处理方式                                    |
| ------------ | --------------------------------------- |
| 文件格式不支持      | 上传前客户端检测并提示"不支持该格式，请上传 PDF/TXT/..."     |
| 文件超过大小限制     | 上传前客户端检测并提示"文件大小超过 50MB 限制"             |
| 知识抽取结果为空     | 文件标记"完成"但知识卡数为 0，提示"未抽取到结构化知识，可手动创建知识卡" |
| 对话中 LLM 服务超时 | 展示"AI 响应超时，请重试"，已生成内容保留                 |
| 并发编辑知识卡      | 后端乐观锁，提交时检测版本冲突，冲突时提示"内容已被他人修改，请刷新后重试"  |
| 删除有子目录的目录    | 提示并引导用户选择：将文件移至父目录 / 指定其他目录             |
| 授权知识库被删除     | 查询时过滤无效知识库，分身访问时自动跳过                    |
