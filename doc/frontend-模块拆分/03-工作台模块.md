# 工作台模块详解

## 1. 个人工作台 (My Workspace)

### 1.1 我的分身 (`/my/avatar`)

**文件**: `frontend/app/my/avatar/page.tsx`

**功能描述**:

- 配置个人数字医生分身
- 设置分身名称、简介、开场白
- 选择沟通风格（正式严谨、亲切温和、简洁高效、详尽细致、自定义）
- 管理标签
- 查看统计数据（接诊数、Token 消耗、知识引用）

**UI 组件**:

- `Card` - 配置卡片和统计卡片
- `Avatar` - 分身头像
- `Input` / `Textarea` - 表单输入
- `RadioGroup` - 沟通风格选择
- `Badge` - 标签展示
- `Button` - 操作按钮

**布局结构**:

### 1.2 我的文件 (`/my/files`)

**文件**: `frontend/app/my/files/page.tsx`

**功能描述**:

- 管理个人知识文件
- 上传文件（PDF、DOCX 等）
- 查看文件解析状态
- 文件分类和搜索

**数据结构**:

```typescript
KnowledgeFile {
  id: string,
  owner_type: 'personal',
  file_name: string,
  file_type: string,
  file_size: number,
  status: 'uploading' | 'parsing' | 'extracting' | 'indexing' | 'done' | 'failed',
  created_at: string
}
```

### 1.3 我的知识卡 (`/my/knowledge-cards`)

**文件**: `frontend/app/my/knowledge-cards/page.tsx`

**功能描述**:

- 管理个人知识卡片
- 创建和编辑知识卡片
- 知识卡片类型：循证（evidence）、规则（rule）、经验（experience）
- 查看引用次数和审核状态

**数据结构**:

```typescript
KnowledgeCard {
  id: string,
  type: 'evidence' | 'rule' | 'experience',
  title: string,
  content: string,
  tags: string[],
  online_status: 'online' | 'offline',
  audit_status: 'pending' | 'approved' | 'rejected',
  reference_count: number
}
```

## 2. 科室工作台 (Department Workspace)

### 2.1 科室数字医生 (`/dept/avatar`)

**文件**: `frontend/app/dept/avatar/page.tsx`

**功能描述**:

- 配置科室公用的数字医生分身
- 与个人分身配置类似，但服务于整个科室
- 显示科室级别的统计数据

**特殊组件**:

- `DeptHeader` - 科室页面头部（包含科室切换器）

**布局**: 与个人分身页面类似，但标题为"科室数字医生"

### 2.2 科室文件 (`/dept/files`)

**功能**: 管理科室共享的知识文件

### 2.3 科室知识卡 (`/dept/knowledge-cards`)

**功能**: 管理科室共享的知识卡片

---

## 3. 机构工作台 (Organization Workspace)

### 3.1 机构数字医生 (`/org/avatar`)

**文件**: `frontend/app/org/avatar/page.tsx`

**功能描述**:

- 配置机构公用的数字医生（如导诊机器人）
- 服务于整个医疗机构
- 显示机构级别的统计数据（数据量更大）

**特殊组件**:

- `OrgHeader` - 机构页面头部

**示例配置**:

- 名称: "Mediverse 智能导诊"
- 简介: "负责医院整体介绍、科室导航、预检分诊及常见问题解答"
- 标签: [全科, 导诊, 医院介绍]

### 3.2 机构文件 (`/org/files`)

**功能**: 管理机构级别的知识文件

### 3.3 机构知识卡 (`/org/knowledge-cards`)

**功能**: 管理机构级别的知识卡片

---

## 工作台模块共同特点

### 权限层级

```
个人 (Personal) → 科室 (Department) → 机构 (Organization)
```

### 数据隔离

- 个人: 仅用户自己可见
- 科室: 科室成员可见
- 机构: 全机构可见
