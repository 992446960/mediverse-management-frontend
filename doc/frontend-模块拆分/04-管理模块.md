# 系统管理模块详解

## 1. 机构管理 (`/admin/organizations`)

**文件**: `frontend/app/admin/organizations/page.tsx`  
**权限**: 系统管理员

### 功能描述

- 查看所有医疗机构列表
- 创建、编辑、删除机构
- 管理机构状态（启用/禁用）

### UI 组件

- `Table` - 机构列表表格
- `Dialog` - 新增/编辑对话框
- `AlertDialog` - 删除确认对话框
- `Badge` - 状态标签
- `Button` - 操作按钮

### 数据结构

```typescript
Organization {
  id: string,
  name: string,              // 机构名称
  code: string,              // 机构编码
  description?: string,      // 描述
  logo_url?: string,         // Logo URL
  status: 'active' | 'inactive',
  created_at: string,
  updated_at: string
}
```

### 表格列

| 列名     | 说明             | 组件            |
| -------- | ---------------- | --------------- |
| 机构名称 | 显示机构名称     | Text            |
| 机构编码 | 显示机构编码     | Text            |
| 描述     | 显示描述（截断） | Text (truncate) |
| 状态     | 启用/禁用        | Badge           |
| 操作     | 编辑/删除按钮    | Button Group    |

### 布局结构

```
┌─────────────────────────────────────────┐
│  标题: 机构管理        [新增机构] 按钮  │
├─────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐│
│  │ 机构名称 │ 编码 │ 描述 │ 状态 │ 操作││
│  ├─────────────────────────────────────┤│
│  │ City Gen │ CG01 │ ... │ 启用 │ 编辑││
│  │ Valley M │ VM02 │ ... │ 启用 │ 删除││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

## 3. 用户管理 (`/admin/users`)

**文件**: `frontend/app/admin/users/page.tsx`  
**权限**: 系统管理员、机构管理员、科室管理员

### 功能描述

- 查看用户列表（支持按机构/科室筛选）
- 创建、编辑、删除用户
- 重置用户密码
- 管理用户角色和状态

### UI 特点

- **左侧边栏**: 组织架构树（机构 → 科室）
- **右侧内容**: 用户列表表格

### 数据结构

```typescript
User {
  id: string,
  org_id?: string,          // 所属机构
  dept_id?: string,         // 所属科室
  username: string,         // 用户名
  real_name: string,        // 真实姓名
  avatar_url?: string,      // 头像
  phone?: string,           // 电话
  email?: string,           // 邮箱
  roles: UserRole[],        // 角色列表
  status: 'active' | 'inactive',
  remark?: string,          // 备注
  created_at: string,
  updated_at: string
}

type UserRole = 'sysadmin' | 'org_admin' | 'dept_admin' | 'user';
```

### 角色说明

| 角色       | 中文名     | 权限范围 |
| ---------- | ---------- | -------- |
| sysadmin   | 系统管理员 | 全平台   |
| org_admin  | 机构管理员 | 所属机构 |
| dept_admin | 科室管理员 | 所属科室 |
| user       | 普通用户   | 个人     |

### 布局结构

```
┌─────────────────────────────────────────┐
│ 组织架构树 │  用户列表                  │
│            │                            │
│ 全部用户   │  标题: 用户管理 [新增用户]│
│            │                            │
│ ▼ 机构1    │  ┌──────────────────────┐ │
│   ▶ 科室1  │  │ 用户 │ 机构/科室 │...││
│   ▶ 科室2  │  ├──────────────────────┤ │
│            │  │ 张三 │ 机构1/科室1 │..││
│ ▼ 机构2    │  │ 李四 │ 机构1/科室2 │..││
│   ▶ 科室3  │  └──────────────────────┘ │
└─────────────────────────────────────────┘
```

### 表格列

| 列名          | 说明                 | 组件            |
| ------------- | -------------------- | --------------- |
| 用户          | 头像 + 姓名 + 用户名 | Avatar + Text   |
| 所属机构/科室 | 机构名 / 科室名      | Text (2 行)     |
| 角色          | 角色标签             | Badge (多个)    |
| 备注          | 备注信息             | Text (truncate) |
| 状态          | 启用/禁用            | Badge           |
| 操作          | 编辑/重置密码/删除   | Button Group    |

## 4. 分身管理 (`/admin/avatars`)

**文件**: `frontend/app/admin/avatars/page.tsx`  
**权限**: 系统管理员

### 功能描述

- 查看所有数字医生分身
- 按类型筛选（通用/专科/专家）
- 管理分身状态

### 数据结构

```typescript
Avatar {
  id: string,
  type: 'general' | 'specialist' | 'expert',
  org_id?: string,          // 通用分身所属机构
  dept_id?: string,         // 专科分身所属科室
  user_id?: string,         // 专家分身所属用户
  name: string,             // 分身名称
  avatar_url?: string,      // 头像
  bio?: string,             // 简介
  tags: string[],           // 标签
  greeting?: string,        // 开场白
  style?: string,           // 沟通风格
  status: 'active' | 'inactive',
  created_by: string,
  created_at: string,
  updated_at: string
}
```

### 分身类型

| 类型       | 中文名   | 归属 |
| ---------- | -------- | ---- |
| general    | 通用分身 | 机构 |
| specialist | 专科分身 | 科室 |
| expert     | 专家分身 | 个人 |
