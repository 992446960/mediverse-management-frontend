# 发送消息流程 · 浏览器验证报告

**需求简述**：发送消息流程跑通——消息与附件正确渲染、回复渲染、Markdown 解析、三种发送场景快照。

**目标页面/路由**：`http://localhost:3000/chat`（或 `/chat/session/:id`）

**验证时间**：2025-03-18

---

## 代码修改摘要（已实现）

1. **发出消息与附件渲染**
   - `stores/chat.ts`：发送时为用户消息构建 `parts`，包含 `text` + 每个附件的 `file`/`image` part（`file_name`、图片 `url` 为 `URL.createObjectURL(file)`）。
   - `MessageList.vue`：列表项增加 `fileParts`，在气泡内用 `BubbleRenderer` 渲染文本，用 `a-image`/文件名标签渲染附件（图片缩略图 + 非图片文件名）。

2. **回复消息渲染**
   - 回复仍由同一 `MessageList` + `BubbleRenderer` 渲染，流式结束后展示完整内容。

3. **返回内容 Markdown 解析**
   - `BubbleRenderer.vue`：对 `marked.parse` 做健壮处理——`content` 转为字符串、`try/catch`、`highlight` 内层 `try/catch`、若解析结果非字符串则回退为转义后的纯文本，避免未解析的 md 符号或崩溃。

- **交互体验**：输入框、发送按钮、流式回复与停止加载按钮行为正常；发送后用户消息立即出现在气泡中。

## 手动验证建议（步骤 2、3）

1. **仅附件**：在输入框旁点击回形针，选择一张图片或一个 PDF，不输入文字，点击发送。预期：用户气泡中仅显示附件（图片为缩略图，文档为文件名）。
2. **文字+附件**：输入一段文字，再点击回形针添加 1 个或多个附件，点击发送。预期：用户气泡中先显示文字，下方或同区域显示附件列表/缩略图。
