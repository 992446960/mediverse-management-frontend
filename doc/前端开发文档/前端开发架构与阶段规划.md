## 一、前端开发阶段划分

基于产品优先级（P0 优先）、模块依赖关系、组件复用度三个维度，将前端开发拆分为 **6 个阶段**。每个阶段具备独立可演示成果。

### 阶段总览

```
Phase 1 ──► Phase 2 ──► Phase 3 ──► Phase 4 ──► Phase 5 ──► Phase 6
基础设施     管理后台     工作台核心    数字医生对话   知识搜索      增强完善
(M)          (M)         (XL)         (L)          (M)          (S)
```

| 阶段 | 名称 | 目标 | 优先级 | 工作量 | 依赖阶段 |
|------|------|------|--------|--------|----------|
| Phase 1 | 工程基础设施与认证 | 搭建可运行的工程骨架，完成登录认证 | P0 | M | 无 |
| Phase 2 | 管理后台 CRUD | 完成机构/科室/用户/分身四大管理模块 | P0 | M | Phase 1 |
| Phase 3 | 工作台核心 | 完成三级工作台（文件管理+知识卡+分身配置） | P0 | XL | Phase 1 |
| Phase 4 | 数字医生对话 | 完成 SSE 流式对话、评分反馈、对话测试 | P0 | L | Phase 1, Phase 3（分身配置） |
| Phase 5 | 知识搜索与知识卡 | 知识库搜索多轮对话、我的知识卡独立页 | P1 | M | Phase 3, Phase 4（ChatWindow 复用） |
| Phase 6 | 仪表盘与系统完善 | Dashboard、API Token、国际化完善、性能优化 | P1 | S | Phase 1~5 |

> **说明**：Phase 2 和 Phase 3 之间**无强依赖**，可根据团队人力**并行推进**。

### 各阶段交付物清单

| 阶段 | 交付物 |
|------|--------|
| Phase 1 | 可运行的空壳工程、登录/登出流程、主布局骨架、路由守卫、权限框架、主题/国际化切换 |
| Phase 2 | 机构列表 CRUD、科室带目录树 CRUD、用户带目录树 CRUD、分身向导式创建 |

| Phase 6 | 仪表盘统计、API Token 管理、国际化文案完善、无障碍优化、性能调优 |

## 五、关键架构决策

### ADR-001：工作台三级作用域统一 API 设计

**背景**：个人/科室/机构三级工作台功能高度一致，仅作用域不同。

**决策**：知识库 API 统一使用 `ownerType` + `ownerId` 路径参数化，前端封装一层参数化 API 调用。

**正面**：组件逻辑一份代码三处复用，维护成本低。  
**负面**：组件 Props 需要透传 ownerType/ownerId，层级略深。  
**缓解**：使用 Vue provide/inject 在页面层注入 workspace context，组件层自动获取。

### ADR-002：SSE 流式对话使用 fetch + ReadableStream

**背景**：对话响应使用 SSE 流式传输。

**决策**：使用原生 `fetch` API + `ReadableStream` 手动解析 SSE 事件，而非 EventSource。

**正面**：支持 POST 请求体、自定义 Header（JWT）、手动中止（AbortController）。  
**负面**：需自行实现 SSE 事件解析。  
**理由**：EventSource 仅支持 GET 请求，无法携带 Authorization Header。

### ADR-003：页面组件采用薄壳模式

**背景**：`/views/` 下的页面组件应尽量简单。

**决策**：页面组件仅作为"胶水层"，负责注入 ownerType/ownerId 等上下文参数，核心逻辑全部下沉到 `src/components/` 或 `src/composables/`。

**正面**：逻辑复用最大化，页面文件极简。  
**负面**：组件目录可能较深。  
**缓解**：通过清晰的命名和目录结构保持可读性。
