概论：
一个机构/部门/用户对应只能创建一个并且是一一对应的全科分身/专科分身/专家分身。所以对应的角色都有对应的分身路由和展示对应的工作台菜单。对应的角色只能通过自己角色的工作台去设置自己的分身。
| 角色 | 展示路由   |
| 用户 | 个人工作台 |
| 用户、科室| 个人、科室工作台 |
| 用户、科室、机构| 个人、科室、机构工作台 |
| 用户、机构| 个人、机构工作台 |
| 用户、系统管理员| 个人、科室、机构工作台 |
| 用户、科室、系统管理员| 个人、科室、机构工作台 |
| 用户、科室、机构、系统管理员| 个人、科室、机构工作台 |
系统管理员三个工作台都存在，所用用户只能通过通过不同的路由菜单进入设置全科分身/专科分身/专家分身。
1、分身模块，关于title，我的工作台进入为我的数字分身，科室工作台进入为科室数字医生，机构为机构数字医生，可以通过枚举配置，方便以后维护修改。
2、关于机构选择器使用三联选择器，先按照每个节点的avatar_id字段非空进行赛选，再统一先按照角色最高权限进行数据筛选(详细见2.1)。根据当前进入的路由再次进行数据筛选（详细见2.2）。进入分身配置路由，默认选中到最后一级，根据最后一级所在的对象中avatar_id，请求分身详情和接口进行回显。更改选择器数据时候重新请求回显（目前只有系统管理员会涉及这一操作，其他角色三级选择器为禁用）：

2.1 数据筛选：例如最高角色为：
- 用户 ： 通过登录接口返回的数据中的user.id为参数筛选将tree数据筛选后只保留当前账户对应的用户和当前用户所属的部门和当前部门所属的机构。（三维数组）
- 部门：通过登录接口返回的数据中的user.dept_id为参数筛选将tree数据筛选后只保留当前账户所属的部门下的所有用户和当前账户所属部门和当前部门所属的机构。（三维数组）
- 机构：通过登录接口返回的数据中的user.org_id为参数筛选将tree数据筛选后只保留当前账户所属的机构和当前机构下的所有部门数据和所有部门下的所有用户数据。（三维数组）
- 系统：不用筛选数据。
2.2 当前进入的路由：
- 个人工作台——分身配置：通过登录接口返回的数据中的user.id为参数筛选将2.1 数据后的数据再次进行数据筛选，只保留当前账户对应的用户和当前用户所属的部门和当前部门所属的机构。若当前角色最高角色为系统管理员，不进行数据筛选。（三维数组）
-  部门工作台——分身配置：通过登录接口返回的数据中的user.dept_id为参数筛选将2.1 数据后的数据再次进行数据筛选，只保留当前账户所属的部门和当前部门所属的机构。若当前角色最高角色为系统管理员，将原始完整的tree数据只保留到部门数据。（二维数组）
- 机构工作台——分身配置：通过登录接口返回的数据中的user.org_id为参数筛选将2.1 数据后的数据再次进行数据筛选，只保留当前账户所属的机构。若当前角色最高角色为系统管理员，将原始完整的tree数据只保留到机构数据。（一维数组）
个人工作台——分身配置——三联选择器默认选中机构、科室、用户，不可修改（此时进入的角色只能是用户和系统管理员，用户角色数据经过筛选应该只剩下一组数组，系统管理员不用清理数据，但是次数的选择器必须选到最后一层，也就是必须选到科室才会请求分身的详情数据）
2.3 对tree接口进行改造，每个节点增加一个avatar_id字段，通过该字段调用分身详情接口`GET /api/v1/avatars/{id}`获取分身数据进行回显
