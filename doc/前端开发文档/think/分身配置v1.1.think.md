概论：
一个机构/部门/用户对应只能创建一个并且是一一对应的全科分身/专科分身/专家分身。所以对应的角色都有对应的分身路由和展示对应的工作台菜单。对应的角色只能通过自己角色的工作台去设置自己的分身。
| 角色 | 展示路由   |
| 用户 | 个人工作台 |
| 用户、科室| 个人、科室工作台 |
| 用户、科室、机构| 个人、科室、机构工作台 |
| 用户、机构| 个人、机构工作台 |
| 用户、系统管理员| 不展示，仅开发环境展示 |
1、分身模块，关于title，我的工作台进入为我的数字分身，科室工作台进入为科室数字医生，机构为机构数字医生，可以通过枚举配置，方便以后维护修改。
2、关于数据回显：
GET /api/v1/my/avatar/stats/{owner_type}/{owner_id}
{owner_type} 取值为 personal / dept / org，{owner_id} 对应用户信息中的 user_id / dept_id / org_id

基于一一对应原则，WorkspaceScopeSelector 组件不再需要。每个角色只能操作自己所属的机构/科室/个人分身，不需要在页面内切换机构或科室。科室/机构工作台的 ownerId 直接从 auth store 读取当前用户的 `org_id` / `dept_id`。
