### 关键设计要点

- 布局：顶部导航栏 + 左侧主内容区（约 2/3）+ 右侧知识卡边栏（约 1/3）
- PDF 双模式：仅 PDF 支持「解析视图」「原文视图」切换；其他类型仅预览原文件
- 复用组件：PageHead、a-tabs、a-card、a-tag、app-container
- 支持类型：PDF、xlsx、docx、文本型（txt、md、json、jsonl、csv）
- 渲染方案：pdf/docx/xlsx → vue-office；md → marked；txt/json/jsonl/csv → 自实现

### 3.2 布局结构（引用 Plan 09 3.5）

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 顶部导航栏（返回 | 文件名+副标题 | 状态Badge+原文件下载）           │
├───────────────────────────────────────┬─────────────────────────────────┤
│ 左侧主内容区（flex: 2）                │ 右侧边栏（flex: 1）                │
│ 文件内容预览                           │ 抽取知识卡 (N)                    │
│ [解析视图] [原文视图] ← 仅 PDF 显示      │ 卡片列表                          │
│ 内容渲染区（PdfViewer/DocxViewer/      │                                  │
│  ExcelViewer/TextFileViewer/          │                                  │
│  ParsedDocViewer）                    │                                  │
└───────────────────────────────────────┴─────────────────────────────────┘
```

### 3.3 按文件类型渲染策略

| file_type | 解析视图 Tab | 原文视图 | 实现方式 |
|-----------|-------------|----------|----------|
| pdf | 显示 | PdfViewer | @vue-office/pdf；解析视图用 ParsedDocViewer（marked） |
| docx | 不显示 | DocxViewer | @vue-office/docx |
| xlsx | 不显示 | ExcelViewer | @vue-office/excel |
| md | 不显示 | TextFileViewer | marked 渲染 |
| txt, json, jsonl, csv | 不显示 | TextFileViewer | fetch 后原样展示（自实现） |

### 3.4 知识卡类型映射

| API type | 展示文案 |
|----------|---------|
| evidence | 循证卡 |
| rule | 规则卡 |
| experience | 经验卡 |

### 3.5 状态 Badge 与副标题

- 副标题格式：`{工作区名} · {created_at 格式化日期}`
- 状态 Badge：done→「已解析」，failed→「解析失败」，其他→「处理中」
- 原文件下载：使用 `file_url` 新窗口打开或触发下载
