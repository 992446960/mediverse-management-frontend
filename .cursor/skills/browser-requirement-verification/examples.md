# 需求浏览器验证 · 示例

以下为触发本 skill 后的典型用法示例，便于 Agent 对齐流程与输出格式。

## 示例 1：登录流程验证

**用户需求**：“用浏览器验证登录流程是否正确，并保存关键步骤截图。”

- **目标**：`http://localhost:5173/login`（或项目登录路由）
- **步骤**：
  1. 打开登录页 → 截图 `login-step1-initial.png`
  2. 输入错误账号密码提交 → 截图 `login-step2-error.png`
  3. 输入正确账号密码提交 → 截图 `login-step3-success.png`，并确认跳转到预期页面
- **快照目录**：`doc/e2e-snapshots/`
- **报告**：步骤表 + 效果/逻辑结论 + 快照清单

## 示例 2：知识库搜索交互

**用户需求**：“验证知识库页面的搜索框交互和结果展示，并留完整快照。”

- **目标**：知识库列表或搜索页 URL
- **步骤**：
  1. 进入页面 → 截图初始态
  2. 在搜索框输入关键词并回车/点击搜索 → 截图结果列表
  3. 清空或修改关键词再搜 → 截图更新结果（或空态）
- **快照**：每步一张截图；可选将某步 `browser_snapshot` 存为 `kb-search-step2-snapshot.md`

## 示例 3：表单提交与列表刷新

**用户需求**：“新增一条数据后，列表要正确刷新并展示新项，请用浏览器验证并保存快照。”

- **目标**：对应新增表单页 + 列表页
- **步骤**：
  1. 打开列表页 → 截图当前列表
  2. 进入新增表单，填写并提交 → 截图提交成功态（Toast/提示）
  3. 返回或停留在列表 → 截图列表，确认新项出现且数据正确
- **逻辑验证**：提交后列表数据、排序、分页等是否符合预期
